## 前置準備

### 步驟 1：確認已安裝 Node.js
1. 開啟 PowerShell 或命令提示字元
2. 輸入以下指令檢查：
   ```powershell
   node --version
   npm --version
   ```
3. 如果顯示版本號碼（例如 `v18.x.x`），表示已安裝 ✅
4. 如果沒有，請到 [Node.js 官網](https://nodejs.org/) 下載並安裝

---

## 啟動專案

### 步驟 2：進入專案目錄
在 PowerShell 中執行：
```powershell
cd C:\11_Cursor\lifestory
```

### 步驟 3：安裝依賴套件
```powershell
npm.cmd install
```
> 💡 **提示**：如果遇到 PowerShell 執行原則錯誤，使用 `npm.cmd` 代替 `npm`

**預期結果**：
- 會看到 `added XXX packages` 的訊息
- 安裝過程可能需要 1-2 分鐘

### 步驟 4：啟動開發伺服器
```powershell
npm.cmd run dev
```

**預期結果**：
- 終端會顯示類似以下的訊息：
  ```
  VITE v5.x.x  ready in XXX ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
  ```
- 記下這個網址（通常是 `http://localhost:5173`）

### 步驟 5：開啟瀏覽器
1. 開啟你的瀏覽器（Chrome、Edge、Firefox 等）
2. 在網址列輸入：`http://localhost:5173`
3. 應該會看到「人生小傳 MVP」的歡迎頁面 ✅

## 取得 OpenAI API Key

### 步驟 6：前往 OpenAI 網站
1. 開啟瀏覽器，前往：https://platform.openai.com/
2. 如果還沒有帳號，點擊「Sign up」註冊
3. 如果已有帳號，點擊「Log in」登入

### 步驟 7：建立 API Key
1. 登入後，點擊右上角的個人頭像
2. 選擇「View API keys」或「API keys」
3. 點擊「Create new secret key」
4. 輸入一個名稱（例如：`LifeStory MVP`）
5. 點擊「Create secret key」
6. **重要**：立即複製這個 Key（格式類似：`sk-...`）
   - ⚠️ 這個 Key 只會顯示一次，關閉後就看不到了
   - 如果忘記了，需要重新建立一個新的

### 步驟 8：確認帳號有餘額
1. 在 OpenAI 網站中，點擊「Billing」或「Usage」
2. 確認帳號有足夠的餘額（至少 $5 美元）
3. 如果沒有，點擊「Add payment method」新增付款方式

## 使用流程

### 階段一：歡迎頁面

#### 步驟 9：輸入 API Key
1. 在歡迎頁面，找到「OpenAI API Key」輸入框
2. 貼上你剛才複製的 API Key（格式：`sk-...`）
3. 確認輸入正確
4. 點擊「開始訪談」按鈕

> 💡 **提示**：
> - API Key 會自動保存在瀏覽器的 LocalStorage
> - 下次使用時不需要重新輸入
> - 但請注意：前端直接使用 API Key 僅適合個人測試

---

### 階段二：語音對談訪談流程

#### 步驟 10：開始語音對談
1. 進入訪談頁面後，系統會**自動播放第一個問題**
2. 你會看到對話式的介面，問題會以藍色氣泡顯示
3. 聽到問題後，準備開始回答

#### 步驟 11：使用語音回答
1. 點擊「**開始語音回答**」按鈕（藍色按鈕，有麥克風圖示）
2. 瀏覽器會要求麥克風權限，請點擊「**允許**」
3. 按鈕會變成紅色「**停止錄音**」，表示正在錄音
4. 開始自然地說話，就像在跟朋友聊天一樣
5. 你的語音會即時轉換成文字，顯示在對話中（綠色氣泡）
6. 說完後，點擊「**停止錄音**」結束錄音

#### 步驟 12：語音功能說明
- **自動播放問題**：每進入新問題時，系統會自動用語音讀出問題
- **重新播放問題**：如果沒聽清楚，點擊「重新播放問題」按鈕
- **語音識別**：支援繁體中文語音識別，會即時顯示識別結果
- **手動編輯**：如果語音識別有誤，可以在下方文字框手動修改

#### 步驟 13：繼續回答問題
- 完成回答後，點擊「**下一題**」按鈕
- 系統會自動播放下一個問題
- 重複步驟 11-12，逐一回答所有問題
- 總共有 **16 道問題**，分為 6 個章節：
  1. **開場與現在的自己**（2 題）
  2. **童年與成長背景**（3 題）
  3. **職涯與成就**（3 題）
  4. **人生轉折點**（3 題）
  5. **價值觀與信念**（3 題）
  6. **給家人的話**（2 題，選填）

#### 步驟 14：使用導航功能
- **上一題**：點擊左下角「上一題」返回修改
- **查看所有回答**：點擊右上角「查看所有回答」跳到總覽頁面
- **進度追蹤**：隨時查看頂部的進度條
- **對話記錄**：所有問答都會以對話氣泡的方式顯示，方便回顧

#### 步驟 15：完成最後一題
1. 回答完第 16 題後
2. 「下一題」按鈕會變成「**完成訪談**」
3. 點擊「完成訪談」進入回答總覽頁面

#### 💡 語音對談小技巧
- **環境安靜**：在安靜的環境中使用，語音識別會更準確
- **說話清晰**：說話時清晰、自然，不需要太快
- **分段回答**：如果回答很長，可以分段錄音，系統會自動合併
- **檢查文字**：錄音後檢查識別的文字是否正確，可以手動修改
- **瀏覽器建議**：建議使用 Chrome、Edge 或 Safari 瀏覽器，語音功能支援最佳

---

### 階段三：回答總覽

#### 步驟 15：查看所有回答
1. 你會看到所有章節和問題，以手風琴（可展開/收合）的方式呈現
2. 點擊章節標題可以展開/收合該章節
3. 每個問題下方都有你的回答

#### 步驟 16：編輯回答（可選）
1. 找到想要修改的問題
2. 直接在文字框中編輯
3. 修改會自動保存
4. 也可以點擊「前往此題」回到訪談流程的該題目

#### 步驟 17：檢查完整性
- 系統會檢查是否有必填問題未回答
- 如果有，會顯示黃色警告訊息
- 即使有未回答的問題，仍可以生成傳記（但建議先完成）

#### 步驟 18：生成傳記
1. 確認所有回答都滿意後
2. 點擊底部的「根據這些回答生成人生小傳」按鈕
3. 系統會開始呼叫 OpenAI API

---

### 階段四：傳記生成

#### 步驟 19：等待生成
1. 你會看到載入動畫和「AI 正在整理你的故事…」的訊息
2. 生成過程通常需要 **10-30 秒**
3. 請耐心等待，不要關閉瀏覽器

#### 步驟 20：查看生成的傳記
1. 生成完成後，會顯示「你的個人小傳（草稿）」
2. 傳記會以段落方式呈現，方便閱讀
3. 可以向下捲動查看完整內容

#### 步驟 21：使用傳記功能
在傳記下方有三個按鈕：

1. **重新生成**
   - 使用相同的回答，讓 AI 重新撰寫
   - 每次生成的內容可能會不同
   - 可以多次嘗試，直到滿意為止

2. **回到回答總覽並修改內容**
   - 如果對傳記不滿意，可以返回修改回答
   - 修改後再重新生成

3. **複製全文到剪貼簿**
   - 一鍵複製整篇傳記
   - 複製成功後按鈕會顯示「已複製！」
   - 可以貼到 Word、Google Docs 等文書軟體中

## 功能說明

### 📊 進度追蹤
- 頂部進度條顯示：已完成 X / 總題數
- 視覺化進度條讓你清楚知道完成度

### 💾 自動保存
- 所有回答都會自動保存
- 即使關閉瀏覽器，下次開啟時回答仍然存在
- 但 API Key 需要重新輸入（基於安全考量）

### 🔄 靈活導航
- 可以在任何時候跳轉到任意題目
- 可以從總覽頁面直接跳到特定題目
- 支援「上一題」和「下一題」導航

### ✏️ 隨時編輯
- 在訪談流程中可以隨時修改回答
- 在總覽頁面可以批量查看和編輯
- 修改後可以重新生成傳記

## 常見問題

### Q1：npm 指令無法執行？
**問題**：出現「因為這個系統上已停用指令碼執行」的錯誤

**解決方法**：
- 使用 `npm.cmd` 代替 `npm`
- 例如：`npm.cmd install`、`npm.cmd run dev`

### Q2：開發伺服器無法啟動？
**檢查清單**：
1. 確認已執行 `npm.cmd install` 且成功
2. 確認沒有其他程式占用 5173 埠
3. 檢查終端是否有錯誤訊息

### Q3：API Key 無效？
**可能原因**：
1. API Key 輸入錯誤（檢查是否有空格）
2. API Key 已過期或被撤銷
3. 帳號沒有餘額

**解決方法**：
1. 重新複製 API Key 並貼上
2. 到 OpenAI 網站檢查帳號狀態
3. 確認帳號有足夠餘額

### Q4：生成傳記失敗？
**可能原因**：
1. API Key 無效
2. 網路連線問題
3. OpenAI API 服務暫時無法使用
4. 帳號餘額不足

**解決方法**：
1. 檢查網路連線
2. 重新生成（點擊「重試」或「重新生成」）
3. 到 OpenAI 網站檢查服務狀態
4. 確認帳號餘額

### Q5：回答消失了？
**可能原因**：
1. 清除了瀏覽器的 LocalStorage
2. 使用了無痕模式
3. 換了瀏覽器或裝置

**解決方法**：
- 回答是保存在瀏覽器的 LocalStorage 中
- 如果清除了瀏覽器資料，回答會消失
- 建議在完成後立即複製傳記內容

### Q6：可以跳過某些問題嗎？
- 標記為「選填」的問題可以跳過
- 必填問題建議都要回答，才能生成完整的傳記
- 即使有未回答的問題，仍可以生成傳記

### Q9：語音識別不準確怎麼辦？
**可能原因**：
1. 環境噪音太大
2. 麥克風品質不佳
3. 說話太快或不清楚
4. 瀏覽器不支援

**解決方法**：
1. 在安靜的環境中使用
2. 說話時清晰、自然
3. 可以手動在文字框中編輯識別結果
4. 使用 Chrome、Edge 或 Safari 瀏覽器

### Q10：無法使用語音功能？
**檢查清單**：
1. 確認瀏覽器支援語音功能（Chrome、Edge、Safari）
2. 確認已允許瀏覽器使用麥克風權限
3. 檢查系統的麥克風設定是否正常
4. 如果還是不行，可以使用文字輸入方式回答

### Q11：語音播放沒有聲音？
**可能原因**：
1. 系統音量被關閉
2. 瀏覽器標籤頁被靜音
3. 語音功能不支援

**解決方法**：
1. 檢查系統音量設定
2. 檢查瀏覽器標籤頁是否有靜音圖示
3. 可以閱讀文字問題，不一定要聽語音

### Q7：生成的傳記不滿意怎麼辦？
1. 點擊「重新生成」讓 AI 重新撰寫
2. 點擊「回到回答總覽並修改內容」，修改回答後再生成
3. 可以多次嘗試，直到滿意為止

### Q8：費用如何計算？
- 使用 OpenAI API 會產生費用
- 每次生成傳記大約消耗 $0.01-0.05 美元（取決於回答長度）
- 可以在 OpenAI 網站查看使用量和費用
